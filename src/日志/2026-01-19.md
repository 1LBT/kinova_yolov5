# Kinova m1n6s200 YOLOv5 抓取任务优化日志
**日期**: 2026-01-19

## 1. 遇到的核心问题
*   **大幅位置误差**: 机械臂在执行抓取移动时，XY 平面误差高达 **6cm** 左右，导致抓取严重偏离。
    *   **原因**: Kinova m1n6s200 (6自由度) 机械臂在垂直向下 (`Pitch=180°`) 姿态下容易遭遇**运动学奇异点 (Singularity)**，导致底层控制器为了避开奇异点而提前停止或无法精确执行路径，产生“失速”现象。
*   **动态抓取需求**: 需要在机械臂移动过程中能根据物体位置变化实时调整轨迹（闭环控制）。
*   **夹爪几何偏移**: 引入倾角抓取后，手腕坐标不再等于指尖坐标，导致抓取点偏移。

## 2. 解决方案与实施

### A. 解决运动学奇异点 (关键)
*   **引入倾角 (`GRASP_TILT_X`)**: 放弃垂直抓取，改为 **俯仰倾斜 15度** (`Pitch=180°+15°`)。这让机械臂关节避开了奇异构型，运动立即变得流畅且精准。
*   **参数配置**: 在文件头部添加了可配置参数 `GRASP_TILT_X = 15.0` 和 `GRASP_YAW = 0.0`。

### B. 提高定位精度
*   **闭环位置修正 (Closed-loop Correction)**:
    *   在到达预抓取点后，增加了一个 **"检查-重试" 循环**。
    *   程序会读取当前的实际机械臂坐标 (`CURRENT_POSE`)，计算与目标的误差。
    *   如果误差超过 `POSITION_TOLERANCE` (5mm)，会自动重发移动指令进行微调，最多重试 3 次。
*   **视觉伺服逼近 (Visual Servo)**:
    *   实现了 `visual_servo_approach` 函数。在机械臂从高处下降的过程中（Phase 1），持续以 5Hz 频率检查物体位置。
    *   如果发现物体移动（偏差 > 1cm），立即动态更新目标坐标。
    *   **盲区控制**: 使用 `BLIND_ZONE_MIN_DIST` 参数控制伺服停止的高度，避免因相机离物体太近失焦而导致数据跳变。

### C. 几何补偿逻辑
*   **手腕坐标反推**: 实现了 `calculate_wrist_target` 函数。根据设定的俯仰角和旋转角，利用向量计算反推手腕应该所在的空间位置。
*   **参数调整**: 虽然实现了复杂的补偿算法，但最终测试发现将 **`GRIPPER_LENGTH` 设为 0.0** 效果最好，消除了不必要的额外偏移。

### D. 交互与安全
*   **手动回 Home**: 在主循环中增加了按 `空格` 或 `h` 键直接返回 Home 点的功能，方便调试中断。
*   **安全校验**: 保留了工作空间半径 (`R < 0.7m`) 和高度限制 (`Z > -0.1m`) 的检查。

## 3. 最终代码状态
当前代码位于 `src/ggcnn/ggcnn_kinova_grasping/scripts/kinova_yolo_grasp.py`。

### 关键参数设置
*   `GRASP_TILT_X = 15.0`  # 俯仰角，关键优化
*   `GRASP_YAW = 0.0`      # 水平旋转角
*   `POSITION_TOLERANCE = 0.005` # 5mm 误差容忍
*   `GRIPPER_LENGTH = 0.0` # 夹爪长度补偿归零

### 执行流程
1.  **Home**: 机械臂回到初始位置。
2.  **视觉伺服下降 (Phase 1)**: 快速下降，同时根据视觉反馈动态修正 XY 坐标。
3.  **盲区进近 (Phase 2)**: 进入相机盲区，锁定最后坐标继续下降到预抓取点。
4.  **静态精修 (Correction)**: 检查实际到达位置，如有误差进行点动修正。
5.  **抓取 (Phase 3)**: 计算最终手腕坐标 -> 下降 -> 闭合夹爪。
6.  **提起**: 提起物体并回到预抓取点。
