# Kinova 机械臂抓取系统使用指南

## 系统概述

本系统基于 **YOLOv5** 视觉识别和 **Kinova m1n6s200** 机械臂，实现了针对特定物体的自动识别与抓取。系统采用了先进的**视觉伺服**和**抗奇异点策略**，能够在复杂的桌面环境下稳定工作。

### 核心特性
*   **YOLOv5 目标检测**: 实时识别物体类别与位置。
*   **视觉伺服 (Visual Servo)**: 在机械臂下降过程中动态修正轨迹，大幅提高抓取精度。
*   **抗奇异点倾角抓取**: 采用 15° 俯仰角进近，彻底解决 6 轴机械臂垂直抓取时的运动学奇异点问题。
*   **闭环位置修正**: 到达预抓取点后自动检测误差并微调。

## 快速开始

### 1. 环境准备
```bash
# 进入工作空间
cd ~/kinova_ws

# 设置环境变量
source devel/setup.bash
```

### 2. 启动基础组件

#### 启动机械臂驱动
```bash
# 终端1
roslaunch kinova_bringup kinova_robot.launch kinova_robotType:=m1n6s200
```

#### 启动相机与 TF
**重要**: 必须使用此 launch 文件，它包含了最新的手眼标定参数和 `yolo_frame` 定义。
```bash
# 终端2
roslaunch ggcnn_kinova_grasping handeye_static_tf.launch
```

#### 启动 YOLOv5 检测节点
(确保您已配置好 YOLO 环境)
```bash
# 终端3
roslaunch yolov5_ros yolov5.launch  # (假设包名为 yolov5_ros，请根据实际情况调整)
# 或者直接运行您的检测脚本
python3 src/yolov5/detect_ros.py
```

### 3. 运行抓取程序 (核心)

推荐使用最新的闭环抓取脚本：

```bash
# 终端4
rosrun ggcnn_kinova_grasping kinova_yolo_grasp.py
```

**操作说明**:
1.  程序启动后，机械臂会自动归位到 Home 点。
2.  终端显示 "按 Enter 开始检测并抓取..."。
3.  确保相机视野内有目标物体（如 `sports ball`）。
4.  按 **Enter** 键，机械臂将开始：
    *   视觉伺服下降
    *   盲区进近
    *   执行抓取
    *   提起物体
    *   自动返回 Home
5.  在任何时候按 **空格** 或 **h** 键，可强制中断任务并返回 Home。

## 参数配置

您可以在 `src/ggcnn/ggcnn_kinova_grasping/scripts/kinova_yolo_grasp.py` 文件的头部直接修改以下关键参数：

### 1. 抓取策略参数
```python
# 俯仰角 (Pitch-like tilt)
# 默认 15.0 度。用于避开垂直奇异点。
GRASP_TILT_X = 15.0

# 水平旋转角 (Yaw)
# 默认 0.0 度。可根据物体摆放角度调整。
GRASP_YAW = 0.0

# 夹爪几何参数
# 默认 0.0。若发现抓取点偏前或偏后，可微调此值 (单位: 米)。
GRIPPER_LENGTH = 0.0
```

### 2. 精度控制参数
```python
# 位置误差阈值
# 默认 0.005 (5mm)。到达预抓取点时，若误差超过此值会自动重试。
POSITION_TOLERANCE = 0.005

# 视觉伺服截止高度
# 默认 0.05 (5cm)。在距离物体 5cm 高度前，一直启用视觉闭环修正。
BLIND_ZONE_MIN_DIST = 0.05
```

## 故障排除

### 1. 抓取位置偏差很大 (>5cm)
*   **原因**: 可能是机械臂遭遇奇异点，或者是 TF 冲突。
*   **解决**:
    *   确保 `GRASP_TILT_X` 至少为 10-15 度。
    *   运行 `rosrun tf tf_echo m1n6s200_end_effector camera_link` 检查 TF 是否稳定。
    *   确保没有其他节点（如 `wrist_camera.launch`）在发布冲突的 TF。

### 2. 机械臂动作卡顿/失速
*   **原因**: 运动规划失败或触及关节限位。
*   **解决**: 尝试改变物体在桌面的位置，或者微调 `GRASP_YAW` 角度。

### 3. 相机画面对不准
*   **原因**: `handeye_static_tf.launch` 中的标定参数不准确。
*   **解决**: 参考 `手眼标定指南.md` 重新进行标定。
